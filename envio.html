Envio de trabalhos

Para enviar um trabalho, selecione o nome do trabalho, a linguagem e os arquivos.

Para as linguagens Pascal e Fortran, os programas devem ser compostos por apenas um arquivo. Para C e C++ qualquer número de arquivos, a menos que haja uma restrição indicada no enunciado do trabalho.

Os arquivos que serão enviados devem ter nomes formados apenas por letras sem acentos, dígitos, pontos, hífens e sublinhados. Eles também devem ter certos sufixos, de acordo com a linguagem:

  • C: .c ou .h
  • C++: .cpp ou .h
  • Pascal: .pas
  • Fortran: .f ou .F
  • PDF: .pdf
Se o trabalho especificar o envio de arquivos PDF juntamente com o código fonte então eles devem ter .pdf como sufixo.

O sqtpm usa o sinal retornado ao término do programa para determinar o estado da execução. Evite definir o valor de retorno da função main do seu programa com valores maiores que 127. Convencionalmente retorna-se 0 quando o programa for bem sucedido.

O programa será avaliado pela verificação do resultado da execução com alguns casos-de-teste. Cada caso-de-teste será executado redirecionando um arquivo para a entrada padrão e redirecionando a saída para outro arquivo, com um comando da forma 

programa <caso.in >execucao.out
e depois a saída será comparada com a saída esperada usando diff 
diff execucao.out caso.out
e usando diff ignorando a formatação 
diff -q -b -B -i -w execucao.out caso.out
Se houver algum caso-de-teste aberto, o sqtpm exibirá a entrada, a saída produzida pelo programa e a saída esperada. Em alguns trabalhos a saída correta não é única. Nesses casos o sqtpm exibirá apenas a entrada e a saída produzida pelo programa para os casos-de-teste abertos.

Não deixe de verificar o número máximo de envios permitidos para o trabalho.
